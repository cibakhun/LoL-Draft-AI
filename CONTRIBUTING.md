# Contributing to VANTAGE

First off, thanks for taking the time to contribute!

The following is a set of guidelines for contributing to VANTAGE (Titan V3.5). These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs

Atomic bug reports are highly anticipated! If you find a bug, please create a GitHub issue including:
*   A descriptive title.
*   Steps to reproduce the issue.
*   Expected vs. actual behavior.
*   Logs or screenshots if applicable.

### Pull Requests

1.  **Fork the repo** and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure your code adheres to standard Python (PEP 8) style guidelines.
4.  Issue that pull request!

## Coding Style

*   **Python**: We follow PEP 8.
*   **Variable Names**: Use `snake_case` for variables and functions, `CamelCase` for classes.
*   **Architecture**: Please maintain the separation between the `engine` (Neural Logic) and `interface` (UI/LCU).
